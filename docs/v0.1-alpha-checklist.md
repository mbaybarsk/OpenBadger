# OpenBadger v0.1 Alpha Checklist

## Release-readiness checklist

- [ ] PostgreSQL migrations apply cleanly on a fresh database
- [ ] Server starts with a valid credential encryption key
- [ ] New credential profiles are stored encrypted at rest
- [ ] Secret-bearing scheduled job payloads are stored encrypted at rest
- [ ] Secrets are redacted from structured logs
- [ ] Collector heartbeats appear in the UI and stale nodes are detected
- [ ] Sensor heartbeats appear in the UI and stale nodes are detected
- [ ] Observation retention deletes raw observations older than the configured policy
- [ ] `/debug/vars` exposes operational counters
- [ ] Installation and first-run documentation is verified in a clean environment
- [ ] Core alpha workflows pass in a lab environment

## Core workflow smoke checks

- [ ] Create a site
- [ ] Enroll one collector
- [ ] Enroll one optional sensor
- [ ] Create SNMP, SSH, or WinRM credential profiles
- [ ] Create at least one target range
- [ ] Create at least one scan profile
- [ ] Create at least one schedule
- [ ] Confirm jobs are created and leased
- [ ] Confirm observations are ingested
- [ ] Confirm assets are visible in inventory
- [ ] Export inventory CSV successfully
